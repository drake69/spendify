## Titolo
`monitor: ETA deve essere del bench completo, non del modello corrente`

## Label
`enhancement` · `P2·v3.0`

## Body

### Problema

L'ETA visualizzata dal monitor (`benchmark/monitor_benchmark.py`) è calcolata
su `avg_dur = media di tutti i file completati` — inclusi i modelli già finiti,
che possono avere velocità molto diverse dai modelli ancora da processare.
Il risultato è un'ETA inattendibile a metà bench: se i primi modelli erano
veloci, l'ETA sottostima il tempo rimanente; se erano lenti, lo sovrastima.

**Comportamento attuale:**
```
ETA = (total_exp - total_done) / rate_fps
rate_fps = 1 / avg_dur_globale   ← media su TUTTI i file già completati
```

**Comportamento atteso:**
L'ETA deve essere calcolata sulla velocità del **modello attualmente in
esecuzione** (rate osservato sugli ultimi file dello stesso `model key`),
proiettata sui file rimanenti dell'**intero bench** (tutti i modelli ancora
da completare × file per modello × fasi).

### Soluzione proposta

1. In `_snapshot()`: calcolare `rate_fps_current` come media degli ultimi
   N file (es. N=10) appartenenti al model key attivo (quello con il
   `counts[key]` più alto e ancora in corso).
2. Usare `rate_fps_current` per l'ETA globale, mantenere `rate_fps_global`
   per il rate storico medio (già utile come informazione secondaria).
3. `total_exp` deve contare correttamente entrambe le fasi
   (classifier + categorizer) per ogni modello quando si usa
   `run_benchmark_full` — cioè `n_models × exp_per_model × 2_phases`.
